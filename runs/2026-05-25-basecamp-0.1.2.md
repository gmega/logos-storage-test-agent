## Notable observations / changes during the run

- `logos-cli.py` install_module helper was extended to recognise both `Install` and
  `Upgrade` confirmation buttons (was Install-only, 5 s timeout → now 30 s).
- The Storage UI's onboarding cards are `OnBoardingCard_QMLTYPE_*` containers; the inner
  `QQuickText` is not clickable. Click the card itself.
- Native file dialog was bypassed by calling
  `backend.callCoreModuleMethod("storage_module", "uploadUrl"/"downloadToUrl", ...)` —
  return values collapse to `null` over the JSON bridge; read the CID from the UI tree
  (Manifests panel) or from `storage.log`.
- CLAUDE.md still has a typo: `logos-storage-ui-module` (404). The real repo is
  `logos-storage-ui`. Existing `tools/install.sh` uses the correct name.
